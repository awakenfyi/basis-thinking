# Dialects — Basis beyond the homepage

The roles keep their jobs; the modules change costume. An asset must account for all four roles — a deliberate omission is declared, never accidental.

## Core dialects

| Domain | TELL | SEE | PROVE | DO |
|---|---|---|---|---|
| **Web page** | hero, H1 | product UI, demo | logos, numbers, stories | CTA tiers T1–T4 |
| **Copy (sentence scale)** | headline | feature line | stat, named customer | the verb |
| **Deck** | POV slide | demo/screens | customer + numbers slides | next-step slide |
| **Email / ad / social** | hook line | image/clip | one receipt | one verb (often Tell+Prove+Do only — declared) |
| **PR** | the narrative | — (often omitted, declared) | third-party validation | the ask behind the story |
| **Brand system** | Tell-dominant; the elevator test is its pass/fail | | | |

## Expansion map — where the grammar goes next

**The pitch.** A fundraise deck is Tell (vision) / See (product) / Prove (traction) / Do (terms). Most failed pitches are a Tell×Inspire stack with the Prove arriving too late — a layer-1 sequencing failure.

**Hiring, both directions.** A job post: Tell (mission) / See (the actual work) / Prove (why us — glassdoor-proof, not adjectives) / Do (apply friction). A resume or portfolio is the same four burdens in reverse; most resumes are all Tell.

**Product surfaces.** Onboarding flows, empty states, pricing pages, changelogs, app-store listings — each is a Basis asset. An empty state is a micro-page: Tell what this space becomes, See an example, Prove it's worth it, Do the first action.

**Docs and developer tools.** A README is pure Basis: the claim (Tell), the screenshot or demo (See), the benchmark or adopter list (Prove), the install one-liner (Do). "Try it in 30 seconds" is a DO-first doctrine.

**Education.** A lesson: hook (Tell), demonstration (See), evidence it matters (Prove), exercise (Do). The diagonal holds; great teachers work the off-diagonal.

**Science communication.** The abstract is proto-Basis: claim / method / results / implications map to Tell / See / Prove / Do almost verbatim. Papers that "don't land" usually bury See.

**Physical spaces.** Packaging shelves (Tell on the front, Prove on the back, Do at the price tag), trade-show booths, museum exhibits, menus, real-estate listings — the grammar predates the web and outlives it.

**Film and media.** A trailer: story (Tell), footage (See), laurels and quotes (Prove), date (Do).

**The AI-native frontier — agent outputs themselves.** An agent's final report is a Basis asset: the finding (Tell), the artifact or diff (See), the receipts — counts, logs, hashes (Prove), the next action (Do). An agent that Tells without Proving is the oldest failure in the book wearing new clothes; grading agent deliverables on four-role completeness is Basis as an output-quality gate for AI. This is the largest expansion surface: every AI system that reports to a human needs this grammar, and almost none have it.

## The test for any new dialect

Ask of the domain: *does a stranger arrive ignorant and need to leave acting?* If yes, the four burdens exist, the diagonal defaults exist, and Basis reads it. If no — a diary, a poem — Basis has no jurisdiction, and saying so is part of the method.
